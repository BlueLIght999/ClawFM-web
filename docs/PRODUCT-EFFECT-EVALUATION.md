# 产品效果评测集

本评测集用于把“AI DJ 电台是否好用”落成可重复执行的离线指标。它不是线上埋点系统，而是第一版产品效果基准：用固定会话事件样本跑出 7 条关键链路、21 个指标，帮助后续重构判断有没有破坏核心体验。

## 运行方式

```bash
cd server
npm run eval:product
```

配套单元测试：

```bash
cd server
npm test -- product-effect-metrics.test.js
```

## 文件

- `server/domain/evaluation/productEffectMetrics.js`：纯指标计算函数，无 IO、无外部服务依赖。
- `server/evaluation/productEffectEvalSet.js`：初始离线样本集，覆盖顺滑开播、冷启动兜底、用户干预恢复。
- `server/evaluation/runProductEffectEvaluation.js`：命令行入口，输出 JSON 报告。
- `server/__tests__/product-effect-metrics.test.js`：TDD 回归测试，锁住指标口径。

## 关键链路与指标

| 链路 | 三个指标 |
| --- | --- |
| 顺利开播 | 开播成功率、开播耗时中位数、冷启动失败兜底率 |
| 连续播放 | 静默中断次数、下一首推进成功率、队列补充成功率 |
| DJ 主持 | DJ 播报启动率、播报完成率、播报阻塞率 |
| 个性化推荐 | 推荐入队率、推荐接受率、推荐新鲜度 |
| 用户干预 | 意图识别命中率、干预执行成功率、干预满意率 |
| 今日计划 | 计划生成率、计划使用率、计划兑现率 |
| 登录与音乐资产 | 登录成功率、歌单加载率、画像构建率 |

## 当前样本输出摘要

初始样本包含 3 个会话、7 条链路、21 个指标。当前注意项主要集中在：

- 连续播放：存在 1 次超过 10 秒的静默中断，队列补充成功率为 `0.667`。
- DJ 主持：播报启动率 `0.667`，完成率 `0.5`，阻塞率偏高。
- 个性化推荐：推荐接受率与新鲜度均为 `0.667`。
- 用户干预：干预执行成功率 `0.667`，干预满意率 `0.5`。
- 登录与音乐资产：画像构建率 `0.667`。

这些数值来自刻意设计的混合样本，目的不是证明产品已经达标，而是让评测集能反映“哪里不好”。

## 下一步
